一、实验目的
分析学生选课系统
使用GUI窗体及其组件设计窗体界面，完成学生选课过程业务逻辑编程，基于文件保存并读取数据，处理异常。
一、系统角色分析及类设计
例如：学校有“人员”，分为“教师”和“学生”，教师教授“课程”，学生选择课程。
定义每种角色人员的属性，及其操作方法。
属性示例：​人员（编号、姓名、性别……）
教师（编号、姓名、性别、所授课程、……）
​学生（编号、姓名、性别、所选课程、……）
​课程（编号、课程名称、上课地点、时间、授课教师、……）
